# Evolution Rules

## memory -> skill
满足全部条件时，允许从 memory 升级为 skill：
- 在同一项目重复出现 >= 2 次，或多个项目都出现
- 有明确触发条件
- 有明确步骤
- 可被重复执行
- 已被验证有效

## skill -> rule
满足全部条件时，允许从 skill 升级为 rule：
- 多个项目可复用
- 已稳定
- 不依赖强业务语义
- 作为规范比作为流程更合适

## Do Not Promote
以下内容不得升级：
- 一次性 workaround
- 强业务耦合技巧
- 仍存在明显争议的做法
- 缺少验证的经验

## Promotion Output
升级时必须说明：
- 来源
- 为什么值得升级
- 适用范围
- 风险
