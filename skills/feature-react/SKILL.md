# When to Use
- 新增 React 页面
- 新增组件或交互
- 新增状态管理逻辑
- 新增查询/突变流程

# Steps
1. 明确功能目标
2. 明确 UI、状态、数据来源
3. 识别应放在 component / hook / store / api 的位置
4. 优先复用已有模式
5. 按层实现
6. 处理加载、错误、空态
7. 验证主流程与关键边界
8. 将项目特有模式写入 project memory

# Output
- 功能拆分
- 关键状态流
- 影响文件
- 验证建议
